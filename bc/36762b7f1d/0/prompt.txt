# Improve Entity Naming in Debug Inspector

## Context

Entities in the debug inspector are poorly named, making it hard to identify what's what during debugging. Walls, tiles, and misc entities all show as "Tile #HASH". NPCs show as "Enemy #HASH" instead of their actual name (e.g., "Boldo"). Child entities like orbs and dialogue UI elements are also indistinguishable. The goal is to make every entity identifiable at a glance.

## Changes

### 1. Add `Name` field to `EntityInfo` (core)

**File:** `MonoDreams/Component/EntityInfo.cs`

Add an optional `Name` parameter alongside the existing `Type`:

```csharp
public class EntityInfo(string type, string name = null)
{
    public string Type { get; } = type;
    public string Name { get; } = name;
}
```

This is backwards-compatible — all existing `new EntityInfo("SomeType")` calls continue to work.

### 2. Update inspector display format

**File:** `MonoDreams.Examples/Inspector/DebugInspector.cs` (lines 79-81)

Change the display name logic to show name + type when both are present:

```
Name present:  "Boldo (NPC) #ABC123"
Name absent:   "Wall #ABC123"
No EntityInfo: "Entity #ABC123"
```

### 3. Fix entity types in factories (Examples)

| Factory | Current `EntityInfo` | New `EntityInfo` |
|---|---|---|
| `WallEntityFactory` | `"Tile"` | `"Wall"` |
| `NPCEntityFactory` | `"Enemy"` | `"NPC"` + name from custom fields |
| `MiscEntityFactory` | `"Tile"` | `"Misc"` (not registered anywhere, consistency fix) |

**Files:**
- `MonoDreams.Examples/EntityFactory/WallEntityFactory.cs` (line 37)
- `MonoDreams.Examples/EntityFactory/NPCEntityFactory.cs` (line 29) — also read `name` from `request.CustomFields` if present
- `MonoDreams.Examples/EntityFactory/MiscEntityFactory.cs` (line 36)

**Safety check:** No system matches on `"Tile"` or `"Enemy"` strings for behavior. `GameCollisionHelper` only matches: Player, Collectible, Projectile, Zone, NPCZone, Obstacle. The runner collision helper matches: Player, Collectible, Obstacle. So these renames are safe.
