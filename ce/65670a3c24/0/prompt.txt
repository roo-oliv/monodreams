# Fix: TileEntityFactory uses hardcoded texture instead of the one from LDtk

## Context

After registering entity factories, the level loads but grass tiles render as tall vertical strips instead of proper tile sprites. Walls render correctly.

**Root cause:** `TileEntityFactory` (line 15, 43) hardcodes `_tilemap = content.Load<Texture2D>("Atlas/Tilemap")` for all tiles. The LDtk parser passes the correct per-layer tileset texture via `request.CustomFields["tilesetTexture"]`, but `TileEntityFactory` ignores it. The source rectangle coordinates (from LDtk) are applied against the wrong texture, producing garbage pixels rendered as vertical strips.

`WallEntityFactory` already handles this correctly — it extracts both `tilesetTexture` and `layerDepth` from custom fields. `TileEntityFactory` needs the same pattern.

## Fix

**File:** `MonoDreams.Examples/EntityFactory/TileEntityFactory.cs`

1. Remove the hardcoded `_tilemap` field (line 15)
2. Extract `tilesetTexture` and `layerDepth` from `request.CustomFields` (same pattern as `WallEntityFactory` lines 40-41)
3. Use them when setting `SpriteInfo` (replace lines 41-50)

```csharp
// Extract from custom fields (same as WallEntityFactory)
var layerDepth = request.CustomFields.TryGetValue("layerDepth", out var depth) ? (float)depth : 0.09f;
var tilesetTexture = request.CustomFields.TryGetValue("tilesetTexture", out var texture) ? (Texture2D)texture : null;

if (tilesetTexture != null)
{
    entity.Set(new SpriteInfo
    {
        SpriteSheet = tilesetTexture,       // was: _tilemap (hardcoded wrong texture)
        Source = new Rectangle((int)request.TilesetPosition.X, (int)request.TilesetPosition.Y,
            (int)request.Size.X, (int)request.Size.Y),
        Size = request.Size,
        Color = Color.White * request.Layer._Opacity,
        Target = RenderTargetID.Main,
        LayerDepth = layerDepth,            // was: 1f (hardcoded)
    });
}
```

The constructor can also drop the `content` parameter since `_tilemap` is removed (or keep it for potential future use — user's preference).
